坑 #1

PDF 同步阻塞 → 异步任务模型

🚨 症状

导入 100+ 题真题，单次 LLM 输出超 8K token 被截断

HTTP 网关 60s 超时直接断开

🔬 根因

把所有题塞一次 LLM 调用太贪婪 — 既慢又脆

🛠️ 解法

  • 按字符切 chunk（1200 字/批）
  • chunk 在 题号边界对齐，无字符丢失
  • setImmediate 后台跑 worker
  • 每个 chunk 做完就写库 → 进度可见
  • 失败时已识别的 candidates 保留
  • 1.5s 轮询，前端展示批次进度
坑 #2

服务重启打断 worker → 断点续传

🚨 真实场景

用户上传 PDF 跑到 chunk 14/41。
我同时部署代码 → systemd restart。

worker 进程被 SIGTERM 杀掉。

selfHealOnBoot 把 running job 标 failed('server_restart')。

用户看到："AI 调用失败"

🛠️ 修复（v0.0.8.2）

  • 把 chunks 数组写进 db (jsonb)
  • worker 从 doneChunks 读出起点
  • candidates 持久化累积
  • selfHealOnBoot 改成 setImmediate 续跑，不再标 failed

启示：进程内 in-memory map 是定时炸弹。
所有 worker 状态必须能从 db 单方面恢复。

坑 #3

章节识别 147 个 · 真实是 7 个

🚨 用户反馈

"我上传一本教材，怎么识别出来 147 章？"

🔬 根因

贪婪正则 [^\n]{0,80} 把"第 N 章"后面 80 字的正文都吞掉，每个 chunk 里都"匹配"出一个新章节。

🛠️ 解法

function cleanChapterTitle(raw) {
  // 只保留 "第 N 章" + ≤20 个中文字符
  const m = raw.match(/第[一二三...]+章[^\n]{0,20}/);
  return m?.[0]?.trim();
}

坑 #4

登录后页面不跳转 · 异步 race

🚨 症状

输入完账号密码点登录，页面不动。手动刷新一下才进去。

🔬 根因

setToken → navigate("/") → RequireAuth 看到 user 还是 null → 弹回登录页。
token 已经写了，但 AuthContext 的 user 还没 refetch 完。

🛠️ 解法

async function login() {
  setToken(jwt);
  await refresh();   // 等 me() 拉完 user
  navigate("/");     // 这时 RequireAuth 才不会弹
}

更深的启示：
"状态写了" ≠ "状态生效"。
所有同步跳转前 await 异步 prerequisite。

function buildStudySystemPrompt(profile, stats) {
  return [
    `你是 ${profile.examName} 备考的 AI 陪学助手。`,
    `今天是 ${cnDate}。`,
    '',
    `下面是用户最新的学习状态：`,
    `- 距离考试：${stats.daysUntilExam} 天`,
    `- 题库总数：${stats.totalQuestions} 道 · 错题本：${stats.wrongbookCount}`,
    `- 近 7 天答题：${stats.attemptsLast7Days} 次`,
    `- 上次答题：${stats.daysSinceLastAttempt} 天前`,
    '',
    `原则：`,
    `- 控制 200 字内`,
    `- 给具体可执行建议 ("今天先刷错题本 10 道")`,
    `- 错题本 > 20 道 → 优先清积压`,
    `- 距考 < 30 天 → 冲刺策略；> 60 天 → 节奏稳定`,
  ].join('\n');
}
